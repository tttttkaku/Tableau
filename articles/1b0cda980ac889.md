# はじめに

Zennの記事をGitHub管理にしようと思い立ち、環境構築を始めたのですが、初手でいくつか詰まったポイントがあったのでメモとして残します。

## 1. GitHubリポジトリのクローンでの失敗

まずはGitHubでリポジトリを作成し、ローカルにクローンしました。

```bash
git clone [https://github.com/tttttkaku/Tableau.git](https://github.com/tttttkaku/Tableau.git)
ここで、ついいつもの癖で .git まで含めて移動しようとしてしまい、エラーに遭遇。

Bash
ntakaku@ntakaku-ltmy8qv ~ % cd Tableau.git
cd: no such file or directory: Tableau.git
.git はURLの一部であって、ローカルに生成されるフォルダ名には付かないんですね。

正解：

Bash
cd Tableau
2. npm error code ENOVERSIONS との遭遇
次に、Zennの初期化をしようと以下のコマンドを打ちました。

Bash
npx zenn init
すると、無情にもこんなエラーが出ました。

Plaintext
npm error code ENOVERSIONS
npm error No versions available for zenn
「zenn」という名前のパッケージ単体は存在しないため、バージョンが見つからないというエラーでした。正解は zenn-cli でした。

3. 解決策：zenn-cliのインストール
しっかり zenn-cli をインストールしてから初期化する必要があります。

Bash
# パッケージ名を正しく指定してインストール
npm install zenn-cli

# その後で初期化を実行
npx zenn init
これで無事に articles フォルダや books フォルダが作成されました！

4. 記事の作成とプレビュー
準備が整ったので、記事の雛形を作成します。

Bash
npx zenn new:article
VS Codeを使っているなら、以下のコマンドで直接開いて執筆を始められます。

Bash
code articles/ファイル名.md
また、執筆中にプレビューを見たいときは以下のコマンドが便利です。

Bash
npx zenn preview
5. まとめ
リポジトリに cd するときは .git は不要。

インストールするのは zenn ではなく zenn-cli。

エラーが出ても、ログを読んでわからなければ AI（Gemini） に助けを求めればなんとかなる！

これからガシガシ記事を書いていこうと思います。
